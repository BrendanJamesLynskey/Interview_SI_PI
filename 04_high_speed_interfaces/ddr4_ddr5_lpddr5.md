# DDR4, DDR5, and LPDDR5

## Overview

DDR (Double Data Rate) SDRAM is the dominant memory technology in servers, desktops, laptops, and embedded platforms. Unlike point-to-point SerDes links, DDR interfaces are parallel, multidrop, and bidirectional — a combination that creates unique signal integrity challenges. Understanding the physical layer constraints, topology trade-offs, and routing rules for DDR4, DDR5, and LPDDR5 is essential for PCB engineers working on CPU, GPU, FPGA, and SoC platforms.

This document covers the fly-by topology and write levelling mechanism, per-signal routing rules, signal grouping conventions, and the VrefDQ reference voltage scheme. DDR5 introduces per-DIMM power management, on-die ECC, and a new sub-channel architecture; LPDDR5 adds further optimisation for mobile power budgets.

---

## Tier 1: Fundamentals

### Q1. What is the fly-by topology, and why is it used for DDR4/5 command and address signals rather than a point-to-point or T-branch topology?

**Answer:**

A fly-by topology routes the command/address (CA) and clock signals as a daisy-chain: the PCB trace passes through (or very near) each DRAM component in sequence before terminating at the far end.

```
Controller ──────[DRAM 0]──────[DRAM 1]──────[DRAM 2]──────[DRAM 3]──── ODT termination
              T-stub 0       T-stub 1       T-stub 2       T-stub 3
```

Each DRAM is connected via a short stub (typically < 5 mm) off the main through-route.

**Why fly-by is used:**

At DDR4 speeds (up to 3200 MT/s, signal rise times < 300 ps) a branched T-topology creates stubs whose electrical length is non-negligible. A 25 mm stub on a PCB with $v_{pd} \approx 167$ ps/mm has an electrical length of ~4 ns round-trip — half the bit period at 3200 MT/s. This stub acts as a resonant transmission line stub, causing a deep notch in the frequency response near:

$$f_{stub} = \frac{v_{pd}}{4 \cdot l_{stub}}$$

For a 25 mm stub: $f_{stub} \approx \frac{1}{4 \times 4\text{ ns}} \approx 62.5$ MHz — far too low.

In contrast, the fly-by topology has no significant stubs on the CA/CLK network, so the signal sees a well-terminated, low-reflection path from controller to the last DRAM. The trade-off is **propagation skew**: each DRAM receives the clock and CA signals at a different time. This skew is corrected by the write levelling and read-DQS-gating training algorithms, making fly-by a practical choice for modern high-speed memory.

**Why DQ signals use point-to-point:** Each DQ and DQS signal runs exclusively from the controller to one specific DRAM die (or one rank), so no multidrop topology is required. These traces can be routed with individual length matching and controlled impedance without the multidrop complication.

### Q2. Explain the write levelling procedure. What problem does it solve, and how is the skew corrected in hardware?

**Answer:**

**The problem:**

The fly-by topology introduces a different propagation delay from the controller to each DRAM along the chain. The clock (CK/CK#) arrives at DRAM 0 first, then DRAM 1, and so on. The write data strobe (DQS), however, is generated by the controller in phase with the controller's local clock, so DQS arrives at all DRAMs at approximately the same time — it is not chained. The result is that the phase difference between CK and DQS is different at every DRAM:

$$\Delta\phi_n = t_{CK,n} - t_{DQS,n}$$

where $t_{CK,n}$ is the time CK arrives at DRAM $n$ and $t_{DQS,n}$ is the time DQS arrives. Without correction, the write setup and hold windows would be violated at DRAMs beyond the first.

**The correction mechanism:**

Write levelling (WL) is a per-byte-lane delay calibration:

1. The controller enables write levelling mode in the DRAM (via Mode Register Set, MRS command).
2. The DRAM samples CK on every rising DQS edge sent by the controller and returns a 1-bit feedback signal on DQ[0] indicating whether DQS is before (0) or after (1) the CK rising edge.
3. The controller increments the DQS delay for that byte lane in fine steps until the feedback transitions from 0 to 1, which marks the rising CK edge.
4. Once the CK-to-DQS edge alignment is found, the controller backs off by half a clock period to centre DQS within the CK cycle.
5. Steps 1–4 are repeated independently for every byte lane.

The delay adjustment is implemented in the memory controller's PHY (physical layer) as a digitally-controlled delay line (DLL-based in DDR4; DFE or PI-based in DDR5 PHYs).

**DDR5 write levelling:** DDR5 introduces an additional complication: each sub-channel (two 8-bit sub-channels per 64-bit bus) has its own independent CA/CS topology and its own write levelling loops. The per-sub-channel training reduces the total DQS-to-CK skew that must be corrected.

**Common mistake:** Confusing write levelling with read DQS centering. Write levelling aligns the DQS edge to the CK edge for write operations. Read DQS centering is a separate training step that positions the read DQS window to correctly sample the returned read data.

### Q3. What are the main signal groups in a DDR4 interface? State their impedance targets and routing strategy for each group.

**Answer:**

DDR4 signals are divided into groups that have different routing requirements:

| Signal Group | Signals | Topology | Impedance | Notes |
|---|---|---|---|---|
| Clock | CK/CK# (differential) | Fly-by daisy-chain | 100 Ω differential | Tightest skew; drives all DRAMs |
| Command/Address | CS#, RAS#, CAS#, WE#, A[17:0], BA[1:0], BG[1:0], ACT#, PAR | Fly-by daisy-chain | 40–50 Ω | Matched to CK stub-by-stub |
| Data Strobe | DQS/DQS# (differential) | Point-to-point, one per byte lane | 100 Ω differential | Must be length-matched to DQ within byte lane |
| Data | DQ[7:0] per byte lane | Point-to-point, one per byte lane | 40–50 Ω | Matched within byte lane |
| Data Mask | DM/DBI# | Point-to-point, one per byte lane | 40–50 Ω | Treated as a DQ signal |
| ODT | On-die termination control | Fly-by (registered path) | N/A (logic level) | Co-routed with CA group |
| VrefDQ | Reference voltage | Static power supply | N/A | 0.55 × VDD decoupled at each DRAM |

**Routing strategies:**

- **CA/CLK group:** Route as a fly-by chain. Stub length from the main route to each DRAM ball must be < 5 mm (DDR4) or < 2.5 mm (DDR5) to minimise stub resonance. Maintain consistent spacing between CK and CA traces to ensure similar propagation delay through any shared routing segments.

- **DQS/DQ groups:** Route each byte lane independently. DQ[7:0] and DM# within a byte lane must be length-matched to within ±25 mil (DDR4) or ±15 mil (DDR5). The DQS pair must be length-matched to the byte lane DQ signals within ±25 mil. Inter-byte-lane skew can be much larger since write levelling compensates between lanes.

- **Differential pairs (CK, DQS):** Intra-pair skew must be < 5 mil. Route CK as a matched differential pair with no splits between the positive and negative traces. Never route a differential pair on different layers without using a well-designed via transition.

### Q4. What is VrefDQ and why is its accuracy critical for DDR4 and DDR5?

**Answer:**

VrefDQ is the reference voltage used by the DRAM's input comparator to determine whether each incoming signal is a logic 1 or 0. The receiver compares the incoming data or address signal voltage against VrefDQ:

- If $V_{in} > V_{refDQ}$: logic HIGH
- If $V_{in} < V_{refDQ}$: logic LOW

**DDR4 VrefDQ:**

In DDR4, VrefDQ is supplied externally as a voltage source derived from VDD through a resistor divider. The specification places VrefDQ nominally at $0.5 \times VDD$ (mid-rail) for the DDR4 data bus. The JEDEC JESD79-4 specification requires VrefDQ accuracy to be maintained within ±1% of VDD (i.e., $\pm$ 12 mV at VDD = 1.2 V). The signal-referenced scheme also includes an on-die VrefDQ level accessible via the Mode Register, allowing the controller to scan the VrefDQ level during training to find the optimum operating point that maximises data eye margin.

**DDR5 VrefDQ changes:**

DDR5 eliminates the external VrefDQ supply. Instead, each DRAM contains an on-die voltage reference generator, which eliminates the PCB routing and noise coupling concerns of the DDR4 external VrefDQ. The DDR5 VrefDQ is programmed via the training algorithm and stored in the CA training register. This improves noise immunity and simplifies board design — one fewer power rail to decouple per memory slot.

**Why accuracy matters:**

The input voltage noise budget for a DDR4 DQ receiver is:

$$V_{noise-budget} = V_{swing} \times AC_{noise-margin-fraction}$$

At DDR4-3200, the AC noise margin is specified at ~65 mV. If VrefDQ drifts by 25 mV from its optimal value, the eye margin erodes by ~25 mV on one side — reducing the total noise margin by ~38%. This degrades BER even before accounting for SSO noise, ISI, or crosstalk.

**PCB decoupling for external VrefDQ (DDR4):** Each DRAM VrefDQ pin requires a 100 nF decoupling capacitor placed within 2–3 mm of the ball, to filter high-frequency switching noise from the data bus coupling back through the VrefDQ supply. The 1/8 VDD AC noise on the VrefDQ rail directly subtracts from the data eye timing margin.

### Q5. Compare DDR4, DDR5, and LPDDR5 at a high level. What are the key differences in data rate, bus width, topology, and operating voltage?

**Answer:**

| Parameter | DDR4 | DDR5 | LPDDR5 |
|---|---|---|---|
| Max data rate | 3200 MT/s (JEDEC); 5600 MT/s (XMP/OC) | 8800 MT/s (JEDEC spec) | 8533 MT/s |
| Bus width per DIMM | 64-bit (+ 8 ECC) | 64-bit split into two 32-bit sub-channels | 16-bit (x16 per die) |
| VDD | 1.2 V | 1.1 V | 1.05 V (WCK at VDDQ 1.05 V) |
| Write data clock | DQS from controller | DQS from controller | WCK (write clock, from controller, 2× data rate) |
| On-die ECC | No | Yes (optional) | No |
| CA training | Per-rank | Per sub-channel | Per channel |
| Topology | Fly-by CA, P2P DQ | Fly-by CA (per sub-channel), P2P DQ | Point-to-point only |
| Max channels per DIMM | 1 × 64-bit | 2 × 32-bit sub-channels | N/A (package-on-package) |
| Ranks per DIMM | 1–4 | 1–2 | 1 (typically) |
| Power management | External VR | PMIC on DIMM | Integrated PMIC in SoC/module |

**Sub-channel architecture (DDR5):** Each DDR5 sub-channel is a fully independent 32-bit bus with its own CA, CLK, DQS, and DQ signals. The controller trains each sub-channel separately. This doubles the number of training loops required but halves the bus width per path, making it easier to maintain signal integrity at the higher data rates. It also allows one sub-channel to operate while the other is in a self-refresh state, improving power efficiency.

**LPDDR5 write clock (WCK):** LPDDR5 replaces DQS with a dedicated write clock WCK running at the full data rate (not half rate as in DDR4/5). WCK is source-synchronous from the DRAM to the controller during reads and from the controller to the DRAM during writes. This eliminates the DQS preamble and postamble transitions that consume power in DDR4/5.

## Tier 2: Intermediate

### Q6. A DDR4-3200 design uses four-DIMM slots in a T-topology for the command/address bus. A colleague suggests this is acceptable because T-branch stubs can be kept short. Evaluate this design choice.

**Answer:**

This is incorrect for DDR4-3200, and the reasoning reveals important nuances.

**Why T-topology is problematic at DDR4-3200:**

The DDR4-3200 bit period is $T_{bit} = 1/(3200 \times 10^6) = 312$ ps. The CA signals are single-data-rate (SDR), so the SDR UI is 625 ps. A T-branch creates two stubs from the junction. For the reflection from a stub to arrive back at the main path before the next bit edge, the round-trip delay through the stub must be less than $UI/2 = 312$ ps.

At a PCB propagation velocity of ~167 ps/mm (Dk ≈ 4), this corresponds to a maximum stub length of:

$$l_{stub} = \frac{312\text{ ps}}{2 \times 167\text{ ps/mm}} \approx 0.93\text{ mm}$$

A 0.93 mm stub is physically unrealisable when accounting for via stubs, escape routing, and DIMM connector pin spacing — even 5 mm stubs reflect back into the next bit window. At DDR4-3200, T-topology stubs cannot be kept short enough to be safe with real PCB constraints.

**The impedance discontinuity at the junction:**

Even if the stub is very short (acting as a lumped capacitive load), four DIMM slots in T-topology creates a significant capacitive discontinuity at each junction. Each DIMM slot connector and PCB stub adds 1–3 pF. Four junctions at 3 pF each = 12 pF of additional capacitance on the CA/CLK net, causing amplitude droop and signal slew degradation.

**What is actually used for multi-DIMM systems:**

JEDEC DDR4 multi-DIMM specifications (UDIMM, RDIMM) mandate fly-by topology with series resistors (22–33 Ω) to dampen reflections from the DRAM ball stubs. RDIMM (registered DIMM) uses a Register Clock Driver (RCD) chip that re-buffers all CA/CLK signals, effectively creating a new fly-by chain within the DIMM and isolating the channel from the host controller stub topology.

**Conclusion:** T-topology for DDR4-3200 CA/CLK is not acceptable. The colleague's suggestion would fail signal integrity validation unless the stubs are < 1 mm, which is impractical. Fly-by is the correct topology.

### Q7. Describe the DDR5 sub-channel architecture. How does it change the PCB routing compared to DDR4?

**Answer:**

**DDR5 sub-channel architecture:**

A DDR5 device presents two fully independent memory sub-channels (SC0 and SC1) on a single DIMM slot connector:

- **Sub-channel A:** 32 data bits (DQ[31:0]) + 4 DQS pairs + CA/CS/CLK for SC0
- **Sub-channel B:** 32 data bits (DQ[63:32]) + 4 DQS pairs + CA/CS/CLK for SC1

Each sub-channel has its own command/address bus, clock pair, and chip-select. Training, write levelling, and read DQS centering are performed independently per sub-channel. The DRAM die is physically partitioned into two 32-bit halves internally.

**PCB routing changes versus DDR4:**

1. **Two separate fly-by chains:** Instead of one fly-by chain for all CA/CLK signals, there are two independent fly-by chains (one per sub-channel). This approximately doubles the number of fly-by routing segments on the CA/CLK layer.

2. **More CA/CLK escape routes:** DDR5 DIMM connectors (288-pin DDR5 vs 288-pin DDR4, same physical size) reassign pins to accommodate the dual sub-channel CA buses. The additional CA nets increase congestion in the area near the memory controller.

3. **Independent termination:** Each sub-channel requires its own ODT and CA termination resistors. On-die termination (ODT) is triggered per sub-channel, and the timing of ODT switching must be accounted for in both sub-channel training loops.

4. **PMIC routing:** DDR5 DIMMs include a Power Management IC (PMIC) on the DIMM itself. The DIMM VDD rail now supplies the PMIC (typically 12 V or 5 V), and the PMIC generates VDD (1.1 V), VDDQ (1.1 V), and other supply rails. This changes the bulk power delivery routing from the motherboard VR to the DIMM edge connector.

5. **No external VrefDQ routing:** The external VrefDQ rail required in DDR4 is eliminated in DDR5. This simplifies the power delivery network layout near the DRAM balls and removes one decoupling network.

**Practical routing impact:** A DDR5 channel with a single DIMM slot requires approximately 30% more CA/CLK routing traces than DDR4 due to the dual sub-channel structure. Layer count for DDR5 server memory routing typically increases by two compared to equivalent DDR4 designs.

### Q8. Calculate the maximum allowable DQ-to-DQS skew for a DDR4-3200 byte lane, and explain how this budget is allocated between PCB routing and on-die sources.

**Answer:**

**DDR4-3200 timing budget:**

The data transfer rate is 3200 MT/s. The bit period is:

$$T_{bit} = \frac{1}{3200 \times 10^6} \approx 312.5 \text{ ps}$$

At double data rate, the UI (unit interval) equals half the bit period, or equivalently the full bit period is also the UI for DDR which transfers on both edges:

For DDR, the data rate of 3200 MT/s means 3200 million transfers per second with data transferred on both edges of DQS. The UI equals the time between successive edges. Each DQS half-period is:

$$T_{DQS-half} = \frac{1}{2 \times f_{DQS}} = \frac{1}{3200 \times 10^6} \approx 312.5 \text{ ps}$$

JEDEC JESD79-4C specifies the write data setup/hold timing parameter $tDS / tDH$ (data setup/hold to DQS). For DDR4-3200 (speed bin):

- $t_{DS} = 25$ ps minimum (data-to-DQS setup at DRAM ball)
- $t_{DH} = 25$ ps minimum (DQS-to-data hold at DRAM ball)

The timing window that PCB and system-level skew must fit within is:

$$\Delta t_{max} = \frac{UI}{2} - t_{DS} - t_{DH}$$

In practice, the PCB routing skew budget for DQ-to-DQS within a byte lane is typically stated in JEDEC specs as $\pm 25$ ps for nominal cases, with the length matching guideline equivalent:

$$\Delta l_{max} = \frac{\Delta t_{max}}{v_{pd}} = \frac{25 \text{ ps}}{167 \text{ ps/mm}} \approx 0.15 \text{ mm} \approx 5.9 \text{ mil}$$

Most DDR4-3200 PCB guidelines state a routing skew limit of ±25 mil (approximately ±0.6 mm). This is a tighter rule derived from the full timing budget analysis — accounting for setup/hold margins, silicon process variation, board-level skew allocation, and simulation margin — not directly from the single calculation above. The ±25 mil figure represents the total allowable routing length mismatch within a byte lane after all system timing contributors have been allocated.

**Budget allocation:**

The total skew budget (e.g., 25 ps) is shared between:

| Source | Typical allocation |
|---|---|
| PCB routing length mismatch | ±10 ps (routing length tolerance ±1 mm = ±10 mil at DDR4 pitch) |
| PCB via-to-via mismatch | ±3 ps |
| Controller PHY output skew | ±5 ps (internal output alignment) |
| Package ball-to-ball skew | ±4 ps (on-package routing) |
| On-die input skew | ±3 ps |
| **Total** | **±25 ps** |

This leaves zero margin with all-worst-case allocation — the reason DDR4-3200 requires simulation-verified routing rather than rules-of-thumb alone.

### Q9. What are the primary differences between LPDDR5 and DDR5 from a board routing perspective?

**Answer:**

LPDDR5 and DDR5 serve different markets (mobile/embedded vs desktop/server), and this drives fundamental routing differences:

**1. Package topology (PoP vs DIMM):**

LPDDR5 is almost exclusively used in Package-on-Package (PoP) or multi-chip module configurations where the DRAM is stacked directly on or mounted adjacent to the SoC. Trace lengths are typically 5–25 mm. DDR5 uses DIMM connectors with trace lengths of 50–200 mm from CPU to DIMM slot.

**2. Point-to-point only:**

LPDDR5 does not support multiple ranks or DIMMs on a single channel. The channel is always point-to-point from SoC to a single DRAM package. There are no fly-by chains; the CA signals route directly from SoC to DRAM. This simplifies routing considerably but provides less flexibility for capacity expansion.

**3. Write clock (WCK) vs DQS:**

LPDDR5 uses a differential WCK signal running at the full data rate (e.g., 8533 MT/s → WCK at 8.533 GHz). WCK must be tightly matched to the DQ byte lane it serves, with intra-pair skew < 2 ps and WCK-to-DQ skew < 3 ps. This is tighter than DDR5 DQS requirements due to the higher frequency.

**4. Operating voltage:**

LPDDR5 operates at 1.05 V (VDDQ). Lower voltage reduces noise margins, making every millivolt of supply noise more significant. The PCB power delivery for LPDDR5 must maintain VDDQ ripple < 30 mV.

**5. CA signal count:**

LPDDR5 uses a 6-bit command/address bus (CA[5:0]) plus CS and CKE signals, rather than the wider DDR5 CA bus. The lower pin count simplifies escape routing from SoC BGA packages.

**6. Impedance targets:**

LPDDR5 DQ/DQS targets 50 Ω single-ended (40–60 Ω tolerance). WCK targets 100 Ω differential. These are more relaxed than DDR5 (40–50 Ω SE) because the shorter trace lengths reduce the sensitivity to impedance mismatch.

## Tier 3: Advanced

### Q10. A DDR5-6400 design fails write levelling training on one byte lane of a 4-DRAM unbuffered channel. The controller reports that the DQS delay required to align byte lane 2 is at the maximum of the PHY's delay range (511 steps × 2 ps/step = 1022 ps). Walk through the systematic debug approach.

**Answer:**

A write levelling delay at the maximum of the PHY range (1022 ps) means the controller cannot find the DQS-to-CK alignment because the CK edge at the DRAM arrives so late relative to DQS that the required DQS delay exceeds the PHY's range.

**Step 1 — Measure the actual CK-to-DQS arrival time difference.**

Using a high-bandwidth oscilloscope (> 20 GHz) or TDR, probe the CK and DQS signals at the DRAM ball (or as close as accessible):

- Record $t_{CK, DRAM2}$ — time of CK rising edge at DRAM 2
- Record $t_{DQS, DRAM2}$ — time of DQS rising edge at DRAM 2
- Compute $\Delta t = t_{CK} - t_{DQS}$

If $\Delta t > 1022$ ps, the write levelling failure is confirmed as insufficient PHY range.

**Step 2 — Check fly-by chain routing lengths.**

The fly-by chain introduces a cumulative delay. For DRAM $n$ in the chain:

$$t_{CK,n} \approx t_{CK,controller} + \sum_{k=0}^{n} l_{segment,k} \times v_{pd}$$

If byte lane 2's DQS routes directly from the controller and the CK fly-by chain passes through DRAMs 0, 1, 2, and 3 before terminating, DRAM 3 can have a CK delay of 150–400 ps beyond DRAM 0 (depending on DIMM geometry).

Check the PCB layout:
- Measure the CK fly-by chain total length from controller to each DRAM stub
- Measure the DQS point-to-point lengths from controller to each DRAM

If byte lane 2's DQS is unusually short (e.g., 30 mm) and DRAM 2 is in the middle of a long fly-by chain (e.g., CK must travel 200 mm to reach DRAM 2), the CK-to-DQS arrival difference would be approximately:

$$\Delta t \approx (200 - 30) \text{ mm} \times 167 \text{ ps/mm} \approx 1190 \text{ ps}$$

This exceeds the 1022 ps PHY range, explaining the failure.

**Step 3 — Verify against the JEDEC write levelling specification.**

JEDEC DDR5 write levelling allows a maximum DQS-to-CK alignment range of $\pm tDQSLmax$ (typically ±1000 ps for DDR5-6400). Compute whether the design's routing geometry is within specification.

**Step 4 — Identify root cause and solutions.**

Root causes and mitigations:

| Root Cause | Evidence | Mitigation |
|---|---|---|
| DQS route too short for this DRAM's position in fly-by chain | DQS length << CK path to this DRAM | Add meanders to DQS trace to increase length |
| CK fly-by chain routed with unnecessary extra length | CK chain has extra meanders or detours | Shorten CK fly-by segment feeding this DRAM |
| Wrong DRAM order in fly-by chain (DRAM 2 placed at end) | DRAM order in chain mismatches board geometry | Re-order DRAMs in fly-by chain to match physical placement |
| PHY delay range insufficient for the topology | Design is at the edge of spec | Request controller PHY with extended WL range, or redesign topology |

**Step 5 — Length equalisation rule.**

The correct routing rule is to ensure the DQS length for each byte lane approximates the CK fly-by path length to that DRAM's position:

$$l_{DQS,n} \approx l_{CK,controller \to DRAM_n}$$

This keeps the required write levelling delay correction within the PHY's range (centred near 50% of the maximum range for best margin).

**Practical outcome:** In most cases the fix is to add meanders to the short DQS trace (byte lane 2) until its length approximately equals the CK fly-by path to that DRAM. This is done by re-routing in the PCB EDA tool and confirming with post-route timing extraction simulation before respinning.

### Q11. Derive the maximum allowable stub length for a DDR5-8000 fly-by DRAM stub, given that the stub resonance frequency must be above 5× the Nyquist frequency of the data signal.

**Answer:**

**Given:**

- DDR5-8000: data rate = 8000 MT/s
- Nyquist frequency: $f_N = \frac{data rate}{2} = 4000 \text{ MHz} = 4 \text{ GHz}$
- Requirement: stub resonance frequency $f_{stub} > 5 \times f_N = 20 \text{ GHz}$
- PCB dielectric constant: $D_k = 3.9$ (typical low-loss laminate)

**Step 1 — Compute PCB propagation velocity.**

$$v_{pd} = \frac{c}{\sqrt{D_k}} = \frac{3 \times 10^{11} \text{ mm/s}}{\sqrt{3.9}} = \frac{3 \times 10^{11}}{1.975} \approx 1.519 \times 10^{11} \text{ mm/s}$$

Converting to ps/mm: $v_{pd} = \frac{1000}{1.519 \times 10^2} \approx 6.58 \text{ ps/mm}$

The propagation delay per unit length is:

$$t_{pd} = \frac{\sqrt{D_k}}{c} = \frac{\sqrt{3.9}}{2.998 \times 10^8 \text{ m/s}} = \frac{1.975}{2.998 \times 10^8} \approx 6.59 \times 10^{-9} \text{ s/m} = 6.59 \text{ ps/mm}$$

**Step 2 — Apply the quarter-wave resonance formula.**

A transmission line stub of length $l$ presents a first series resonance (short-circuit reflection minimum) at:

$$f_{stub} = \frac{v_{phase}}{4 \cdot l} = \frac{1}{4 \cdot l \cdot t_{pd}}$$

**Step 3 — Solve for maximum stub length.**

Setting $f_{stub} = 20 \text{ GHz}$:

$$l_{max} = \frac{1}{4 \cdot f_{stub} \cdot t_{pd}} = \frac{1}{4 \times 20 \times 10^9 \times 6.59 \times 10^{-12}} = \frac{1}{527.2 \times 10^{-3}} \approx 1.90 \text{ mm}$$

**Conclusion:**

The maximum stub length for DDR5-8000 with $5 \times f_N$ clearance on a standard $D_k = 3.9$ laminate is approximately **1.9 mm**.

JEDEC DDR5 routing guidelines specify stubs ≤ 2.5 mm, consistent with this analysis (which uses a conservative 5× Nyquist criterion). For higher-speed DDR5 variants or lower-$D_k$ laminates, the stub length must be reduced further. In practice, the stub includes the via, via pad, escape trace, and BGA ball, so the total effective stub length reaches 1.5–2 mm even before any explicit stub routing — making ultra-short stubs the default DDR5 design rule.

### Q12. A DDR5-6400 system shows random single-bit errors on byte lane 3 at elevated temperature (85°C). The errors are not reproducible at room temperature and do not appear on any other byte lane. Describe a systematic investigation methodology.

**Answer:**

Random single-bit errors on one byte lane at temperature, not at room, indicate a marginal timing or voltage condition that worsens with thermal stress. The investigation should proceed in layers.

**Layer 1 — Characterise the failure signature.**

- **Temperature profiling:** Instrument the system with a thermocouple at the DRAM and measure the exact temperature threshold at which errors first appear. Is it specific to the DRAM die, the PCB stackup, or the SoC?
- **Data pattern sensitivity:** Run memory stress tests with checkerboard, PRBS, and all-ones/zeros patterns. If errors only occur with specific patterns, the failure is ISI or crosstalk dependent (pattern-dependent noise).
- **Margin testing:** Reduce the DDR5 data rate in 200 MT/s steps until errors stop. Record the margin (in MT/s) between the highest error-free rate and the nominal rate. A small margin (< 10%) suggests a marginal design at nominal operating conditions.

**Layer 2 — Thermal mechanisms to investigate.**

| Mechanism | Effect on timing | Effect on byte lane 3 specifically |
|---|---|---|
| PCB laminate Dk increases with temperature | Increases propagation delay; changes write levelling offset | If byte lane 3's DQS/DQ is routed on a different layer or through more laminate, it may be disproportionately affected |
| DRAM on-die timing drift | Reduced $t_{DS}/t_{DH}$ at high temp | All bytes affected equally — but check if this DRAM die has a known derating |
| PCB trace resistance increases (~0.4%/°C for copper) | Increases IR drop on DQ lines; slightly reduces swing | |
| Via barrel expansion with temperature | Negligible for signal integrity at DDR5 speeds | |
| Connector contact resistance drift | Increases with oxidation or thermal cycling | N/A for soldered LPDDR5; relevant for DIMM connectors |

**Layer 3 — Signal integrity measurements.**

Using a high-bandwidth oscilloscope with differential probing or a solder-in probe on DRAM balls:

1. **Capture eye diagrams** on byte lane 3 DQ signals at room temp and 85°C. Measure eye height and width reduction. A reduction > 20% of eye height at temperature indicates insufficient voltage margin.
2. **Measure write levelling delay value** at both temperatures by reading the PHY registers. If the required WL delay changes by > 100 ps between temperatures, the timing budget is being consumed by thermal-induced propagation delay change.
3. **Measure DQS preamble waveform.** A degraded DQS preamble at temperature indicates the receive side gating is marginal.

**Layer 4 — Root cause candidates and fixes.**

| Root Cause | Diagnosis | Fix |
|---|---|---|
| Marginal PCB DQ routing length matching on byte lane 3 | DQ length within lane exceeds ±15 mil at temperature (effective Dk shift widens skew) | Re-route to tighter length matching; < ±10 mil |
| Inadequate VrefDQ on-die tracking at temperature (DDR4 external scheme) | VrefDQ measured at DRAM ball drifts > ±1% between temperatures | Improve VrefDQ decoupling; use bypass capacitors < 1 mm from ball |
| Thermal gradient between SoC and DRAM affecting clock skew | CK-to-DQS timing changes with differential temperature | Improve thermal design; add thermal vias under DRAM |
| Byte lane 3 DQ signals routed near a high-activity switching net | SSO noise couples into byte lane 3 at temperature (higher activity at stress conditions) | Increase spacing from SSO aggressors; add ground stitching vias |

**Step 5 — Re-run WL and read DQS training with extended margin at temperature.**

If the system allows, perform BIOS/firmware-level memory training at elevated temperature to find the true operating point at 85°C. The trained delays should be stored and the system re-validated with these values. If stable training is achievable at 85°C, the issue is a fixed offset; if training itself is unstable at 85°C, the channel margin is genuinely insufficient.

## Quick Reference: Key Terms

| Term | Definition |
|---|---|
| Fly-by topology | Daisy-chain routing of CA/CLK signals through each DRAM in sequence, with far-end termination |
| T-topology | Branched routing where all DRAMs see the same trace; unusable at DDR4/5 speeds |
| Write levelling | Training algorithm that aligns DQS phase to CK rising edge at each DRAM to compensate fly-by skew |
| VrefDQ | Input comparator reference voltage; external in DDR4, on-die in DDR5 |
| UI (Unit Interval) | One bit period; $UI = 1/f_{data-rate}$ for SDR, $UI = 1/(2 \times f_{clock})$ for DDR |
| tDS / tDH | Data setup / data hold time relative to DQS at DRAM ball |
| Sub-channel | One of two independent 32-bit memory channels within a DDR5 DIMM |
| WCK | Write clock in LPDDR5; differential clock at full data rate used instead of DQS |
| ODT | On-Die Termination; programmable termination resistors inside the DRAM and controller |
| PMIC | Power Management IC; used on DDR5 DIMMs to generate VDD/VDDQ from a higher input voltage |
| RCD | Register Clock Driver; re-buffers CA/CLK signals on RDIMM to isolate controller loading |
| Stub resonance | Quarter-wave resonance of a T-branch stub that creates a notch in the frequency response |
| Write levelling range | Maximum delay adjustment available in the PHY delay line for write levelling correction |
